### Few-Shot Examples for Multimodal Annotation Agent

These examples demonstrate how to convert natural language scene descriptions into structured JSON annotations.
Follow the exact format and use only the allowed enum values defined in the schema.

---

#### Example 1: Simple Highway Scene with Static Obstacle

**Input Description:**
"Sunny day, ego vehicle driving on urban expressway. White foam box exists in the right lane ahead, about 15 meters away. Ego vehicle should slow down and slightly move left to avoid it."

**Expected Output:**
{
  "driving_environment": {
    "weather": "sunny",
    "time": "day",
    "visibility": "good",
    "road_type": "Highway"
  },
  "lanes": [
    {
      "lane_id": "Lane1",
      "bbox": [0, 400, 800, 600],
      "is_ego": true,
      "lane_type": "driving_lane",
      "direction": "same_direction"
    },
    {
      "lane_id": "Lane2",
      "bbox": [800, 400, 1600, 600],
      "is_ego": false,
      "lane_type": "driving_lane",
      "direction": "same_direction"
    }
  ],
  "static_objects": [
    {
      "obj_id": "obj_1",
      "bbox": [600, 300, 650, 340],
      "type": "plastic_bag",
      "compressible": "Yes",
      "position": "front_right"
    }
  ],
  "ego_vehicle": {
    "lateral_action": "nudge_left",
    "longitudinal_action": "decelerate",
    "reason": "Avoid foam box in right lane ahead"
  },
  "scene_description": "Sunny day, ego vehicle driving on urban expressway. White foam box exists in the right lane ahead, about 15 meters away. Ego vehicle should slow down and slightly move left to avoid it."
}

**Key Points:**
- Use `nudge_left` for slight lateral movement (not full lane change)
- Use `decelerate` when slowing down is needed
- Static object position is `front_right` relative to ego vehicle
- All required fields present: obj_id, bbox, type for static_objects

### Important Guidelines:

1. **Required Fields (Must Include)**:
   - **DrivingEnvironment**: weather, time, road_type (visibility is optional)
   - **LaneInfo**: lane_id, bbox, is_ego (lane_type, direction are optional)
   - **TrafficSignInfo**: sign_id, bbox, type (content, applicable_lanes are optional)
   - **TrafficLightInfo**: light_id, bbox, color (state, countdown, applicable_lanes are optional)
   - **VehicleObject**: obj_id, bbox, type (orientation, behavior, interaction_with_ego are optional)
   - **StaticObject**: obj_id, bbox, type (compressible, position are optional)
   - **EgoVehicle**: lateral_action, longitudinal_action (reason is optional)

2. **Enum Values Only**: Always use exact enum values from the schema:
   - Weather: "sunny", "rainy", "cloudy", "foggy", "snowy", "overcast"
   - Time: "day", "night", "dawn", "dusk"
   - Road types: "Highway", "Urban_Road", "Rural_Road", "Tunnel", "Bridge", "Parking_Lot"
   - Vehicle types: "sedan", "SUV", "truck", "bus", "motorcycle", "bicycle", "van", "taxi", "police_car", "ambulance", "fire_truck"
   - And all other enum fields as defined in schema

3. **Bbox Format**: All bbox fields must be arrays of 4 floats: [x1, y1, x2, y2]

4. **Optional Fields**: Use `null` in JSON or omit the field entirely if value is unknown

5. **Object IDs**: Use consistent naming convention:
   - Static objects: obj_1, obj_2, obj_3...
   - Dynamic vehicles: vehicle_1, vehicle_2, vehicle_3...
   - Traffic signs: sign_1, sign_2...
   - Traffic lights: light_1, light_2...
   - Lanes: Lane1, Lane2, Lane3...

6. **Ego Vehicle Actions**:
   - `lane_change_left/right`: Full lane change maneuver
   - `nudge_left/right`: Slight lateral movement to avoid obstacles (stay in same lane)
   - `keep_lane`: Stay in current lane without lateral movement
   - Combine with appropriate longitudinal action: accelerate, maintain_speed, or decelerate

7. **Interaction Types** (for vehicles):
   - `follow`: Maintain distance behind leading vehicle
   - `yield`: Give way to priority vehicle or pedestrian
   - `bypass`: Go around obstacle
   - `ignore`: No interaction needed

8. **Common Mistakes to Avoid**:
   - Don't use non-enum values (e.g., don't use "car", use "sedan" or specific type)
   - Don't forget bbox for objects that require it
   - Don't confuse `nudge` (slight offset) with `lane_change` (full lane switch)
   - Don't omit required fields (obj_id, bbox, type for most objects)
